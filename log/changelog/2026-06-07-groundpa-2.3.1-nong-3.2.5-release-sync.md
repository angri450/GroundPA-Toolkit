# GroundPA 2.3.1 sync for Nong 3.2.5

Date: 2026-06-07

## Scope

Synchronized Nong.Toolkit.Net with the Angri450.Nong 3.2.5 command surface after Stage19 literature DSL was promoted into the 82-command Nong CLI surface.

GroundPA remains pure .NET / Nong CLI. No Python package, ToolUniverse package, scraping workflow, or browser automation was introduced.

## Changes

1. Updated Nong.Toolkit.Net plugin metadata from `2.3.0` to `2.3.1`.
2. Updated `README.md`, `README.zh-CN.md`, and `skill.zh` to target Nong 3.2.5+ and the 82-command surface.
3. Updated `literature/SKILL.md` from "Stage19 build required" wording to "Nong 3.2.5+ includes `nong lit`".
4. Updated active PDF, Inspect, and Multimodal runtime prerequisites to Nong 3.2.5+.
5. Updated OCR runtime-bundle wording to expect `Angri450.Nong.OcrRuntime.*` 3.2.5 with Nong 3.2.5.

## NuGet Mirror Reality

Attempted global tool updates:

```powershell
dotnet tool update --global Angri450.Nong.Cli --add-source https://mirrors.huaweicloud.com/repository/nuget/v3/index.json
dotnet tool update --global Angri450.Nong.Cli --version 3.2.5 --add-source https://api.nuget.org/v3/index.json
```

Observed in this environment:

- Huawei source returned success but global `nong --version` remained `3.2.4+6854a00297ff6b21e043ca56e048b3fe07bef6c6`.
- NuGet.org reported it could not find `Angri450.Nong.Cli` version `3.2.5`.
- `dotnet tool list --global` still showed `angri450.nong.cli 3.2.4`.

Therefore GroundPA's 3.2.5+ docs are updated for the intended released command surface, while validation that requires `lit` uses the local Angri450.Nong 3.2.5 release executable until the package sources finish syncing.
