# 发现问题：2.0.0 Nong CLI Skill Layer

日期：2026-06-03

## 审查范围

- 2.0.0 skill 暴露面是否符合当前 `nong` CLI 已实现命令。
- 插件清单、README、skills.sh.json 是否同步。
- 未实现的 PPTX/OCR 是否仍被声明为可用 skill。
- skill-manager / claude plugin 基础校验。

## Findings

### HIGH — `pptx/SKILL.md`, `multimodal/SKILL.md` — 未实现能力仍作为 skill 暴露

当前 `nong` CLI 中 `pptx` 和 `ocr` 命令仍为 stub，但 1.2.0 skill 层会触发 PPTX 与 OCR 工作流，容易让模型调用未实现能力。

### HIGH — `word/excel/chart/diagram/genre/SKILL.md` — 仍以旧 NuGet workspace 模式为主

旧 skill 默认让模型创建临时 .NET 项目并调用包 API。2.0.0 目标是统一走 `nong` CLI，否则 skill 层会绕开 CLI 契约和错误码。

### MEDIUM — `.claude-plugin/plugin.json`, `skills.sh.json`, README — 版本与能力表停留在 1.2.0

插件版本、技能分组、安装说明仍描述 1.2.0/旧 Office 七件套，和当前 2.0.0 实现不一致。

### MEDIUM — 缺少 `inspect` 与 `icons` skill

`nong inspect ...` 和 `nong icons ...` 已是当前 CLI 的 implemented 命令，但 skill 层没有对应入口。

### HIGH — `skill-manager/assets/eval_review.html` — 动态表格使用 `innerHTML`

安全扫描报 `UNSAFE_INNERHTML`。虽然数据来自 eval set，但这是实际可修的高风险模式。

### HIGH — email / skill-manager 示例文本触发安全扫描

示例邮箱、私钥示例、CDN 示例和本机绝对路径会被扫描器识别为高风险。需要改为不会误报的占位写法。

## 剩余风险

安全扫描仍报告若干历史文档中的 `HOME_PATH_REFERENCE` medium 和 Apache license HTTP low。当前扫描退出码为 0，不阻塞打包。后续可单独做历史资料清理。
